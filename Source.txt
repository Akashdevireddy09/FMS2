10	Password encryption	US_Sprint_Web_010	Password encryption – simple Caesar cypher	"Password encryption – simple Caesar cypher

You will be encrypting the password field in the application with the help of Caesar Cypher. 

Information on Caesar Cypher:
When we say Caesar cypher, we will be shifting the password value by some digit to the right.
For example:
Consider that we have alphabets as “ABCDEFGHIJKLMNOPQRSTUVWXYZ” if we try to shift it by 2 the updated alphabets would be “CDEFGHIJKLMNOPQRSTUVWXYZAB”. 
Hence if I have a string say “Dave” that would be stored as “Fcxg” 
Some caveats that you can make use of
You can store number as it is no need to shift the number
Special Character can be stored as it is no need to shift the same.
You can decide by how many digits do you need to shift the alphabets.

Design Document:
User will not be aware of how many digits the password is being stored. That should be done in the service file automatically before storing in the database."
